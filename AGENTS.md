# AGENTS.md — Карта агентов Botik

> Этот файл читается оркестратором перед каждой задачей.
> Определяет роли, зоны ответственности и правила параллельной работы.

---

## Агенты и их зоны

| Агент | Зона | Файлы |
|---|---|---|
| **GUI** | Дашборд, API-миксины | `src/botik/gui/api_*.py`, `webview_app.py`, HTML/JS в миксинах |
| **Core** | Торговая логика, исполнение | `src/botik/core/`, `src/botik/execution/`, `src/botik/runners/` |
| **Data** | Данные, ML, маркетдата | `src/botik/data/`, `src/botik/ml/`, `src/botik/marketdata/`, `src/botik/learning/` |
| **Storage** | БД, хранилище, миграции | `src/botik/storage/`, `src/botik/state/` |
| **Strategy** | Стратегии, риск, позиции | `src/botik/strategy/`, `src/botik/risk/`, `src/botik/position/` |
| **Control** | Telegram, внешние интеграции | `src/botik/control/` |
| **Tests** | Тесты всех модулей | `tests/` |

---

## Правила работы агентов

1. Каждый агент **читает `AGENTS_CONTEXT.md`** перед стартом
2. Каждый агент **пишет свой результат в `AGENTS_CONTEXT.md`** после завершения
3. Агент **не трогает файлы чужой зоны** — только читает для справки
4. Если нужно изменение в чужой зоне — пишет запрос в `AGENTS_CONTEXT.md` → оркестратор решает
5. После работы агент запускает тесты своей зоны: `pytest tests/test_<зона>*.py`

## Протокол оркестратора

### Шаг 1 — Анализ задачи

```
Задача затрагивает > 2 разных зоны  → запускать агентов параллельно
Задача в 1 зоне, оценка < 30 мин   → делать самому без агентов
Задача последовательная (B ждёт A) → делать самому по шагам
```

### Шаг 2 — Перед запуском агентов

1. Прочитать `AGENTS_CONTEXT.md` — нет ли незакрытых зависимостей
2. Определить какие агенты нужны
3. Написать задачу каждому агенту в `AGENTS_CONTEXT.md` секция `## Задания`
4. Запустить агентов через `Agent tool` параллельно (один message — несколько вызовов)

### Шаг 3 — После агентов

1. Прочитать `AGENTS_CONTEXT.md` — результаты каждого агента
2. Проверить конфликты
3. Запустить общие тесты: `pytest`
4. Коммит в git
